# 用語集 (GLOSSARY)

将棋用語とゲーム内用語の対応・簡単な解説をまとめます。将棋に詳しくない協力者や、
実装時に用語のブレを防ぐための参照資料です。

## 将棋の基本用語

| 用語 | 説明 |
|---|---|
| 囲い（かこい） | 玉を守るための駒の配置形。美濃囲い・矢倉・穴熊など |
| 戦形（せんけい） | 対局全体の駒組みや戦い方の分類。囲いだけでなく、居飛車・振り飛車や採用する戦法を含む |
| 戦法（せんぽう） | 序盤から中盤へ向けた具体的な攻め方・駒組みの方針。棒銀、四間飛車、角換わり等 |
| 定跡（じょうせき） | 序盤の指し方として研究・蓄積されてきた手順 |
| 居飛車（いびしゃ） | 飛車を初期位置の筋に置いたまま戦う方針。角換わり、横歩取り、相掛かり等を含む |
| 振り飛車（ふりびしゃ） | 飛車を初期位置から別の筋へ移して戦う方針。四間飛車等を含む |
| 棒銀（ぼうぎん） | 銀を飛車先へまっすぐ進出させ、飛車・銀・歩の力を集めて突破する戦法 |
| 四間飛車（しけんびしゃ） | 飛車を四間へ振って戦う振り飛車戦法。本作の後手用定跡では飛車を4筋へ移す |
| 美濃囲い（みのがこい） | 振り飛車で代表的な囲い。玉を飛車と反対側へ移し、金銀で守る |
| 穴熊（あなぐま） | 玉を盤の隅へ深く入れ、香・金・銀等で固める囲い。本作の穴熊村では四間飛車穴熊を扱う |
| 駒の利き | その駒が現在の位置から動ける、または相手の駒を取れる範囲 |
| 頭金（あたまきん） | 相手玉の正面へ、別の自駒に支えられた金を打つ代表的な詰みの形 |
| 駒落ち（こまおち） | 実力差を埋めるため、上手（弱い側）に対して下手（強い側）が特定の駒を外して対局する形式。飛車落ち・角落ち等 |
| 持ち駒（もちごま） | 相手から取った、盤上に再度打てる駒 |
| 成り（なり） | 駒が敵陣に入った際、裏返って能力が変化すること |
| 詰み（つみ） | 玉がどのように応じても捕獲を避けられない状態。対局の終了条件 |
| 持将棋（じしょうぎ） | 両者の玉が敵陣に入る等して勝敗がつかないまま引き分けとなるルール |
| 入玉宣言（にゅうぎょくせんげん） | 玉の位置・敵陣の駒数・点数等の条件を満たした側が勝ちを宣言するルール。本プロジェクトでは27点法を採用 |
| SFEN | 将棋の局面を表すテキスト形式。チェスのFENに相当 |
| USI | Universal Shogi Interface。将棋エンジンとGUI間の通信プロトコル |
| NNUE | 将棋・チェス等のAIで使われる軽量なニューラルネット型評価関数の方式 |
| HalfKP | NNUEの入力特徴量アーキテクチャの一種。HalfKP256とHalfKP512の評価関数には互換性がなく、対応するWASMが必要 |
| MultiPV | エンジンが最善手だけでなく、上位N個の候補手を同時に出力する機能 |
| searchmoves | USIの`go`コマンドで、エンジンが探索する合法手候補を限定する指定。本作では敵専用戦法定跡と振り飛車の維持に使う |
| 次善手（じぜんしゅ） | 最善手の次に良いとされる手 |

## ゲーム内用語（本プロジェクト固有）

| 用語 | 対応する仕組み | 備考 |
|---|---|---|
| 夜古火姫（やこびひめ） | 将棋の学習を案内する神様・マスコット | 行列式から生まれ、数学者ヤコブ・ヤコビをモデルとする。略称は「やこび姫」 |
| 矢印ガイド | 盤上の推奨手や狙いを矢印・強調マス・解説で示す学習補助 | 定跡ガイドと回数制の最善手ヒントは別機能 |
| 定跡書 | 戦法や囲いを専用盤面と矢印で学ぶ恒久教材 | 消耗せず、入手後は定跡集で再閲覧できる |
| 定跡集 | 入手済み定跡書を選び、講座を再生する画面 | 対局中に使用する定跡ガイドの選択元でもある |
| プレイヤー名 | ゲーム開始時に入力する主人公の名前 | シナリオへ安全なテキストとして差し込み、ブラウザ内に保存する |
| 主人公の内心 | 主人公の状況整理や感情を表す1〜2文の短い思考表示 | 主人公の発話とは区別し、選択肢で示した意思と矛盾させない |
| 棋貨（きか） | 店で消耗品を購入するゲーム内通貨 | 初回勝利、宝箱、サブクエスト、ボス報酬等から入手する |
| 復活の呪文 | セーブデータのエンコード文字列 | ドラクエ由来の呼称。進行状況を英数字文字列化したもの |
| 棋神の巻物（仮） | 最善手ヒントアイテム | `items.json` の `type: "hint"` に対応 |
| 待った | 直前の指し手取り消し機能 | `items.json` の `type: "undo"` に対応。将棋用語としても「待った」で通じる |
| 拠点 | RPG進行上の準備・買い物・装備変更を行う場所 | ティラノスクリプト側のシーンとして実装 |
| 章（チャプター） | RPGのストーリー進行単位 | セーブデータの `chapter` に対応 |
| 村内コース | 各村の入口から通常敵を経てボスへ続く地点選択式マップ | 自由歩行ではなく、`world.json`の地点と接続を画面へ描画する |
| 本道 | 通常敵を順番に倒してボスへ進む勝利必須の経路 | 手前の敵へ勝利すると次地点が解禁される |
| 寄り道 | 本道から分かれる任意経路 | 現行コースでは宝箱を配置し、未回収でもボスへ進める |
| 進行マーカー | 主人公と夜古火姫が本道のどこまで進んだかを示す表示 | 連続して踏破した最後の地点に表示し、順序を飛ばした旧セーブでは最初の未勝利地点を次の目的地とする |
| 敵専用戦法定跡 | 敵が章テーマの戦法・囲いを優先して組むための定跡マスタ | `enemy_openings.json`を正とし、プレイヤー用の定跡書や開始戦形とは区別する |
| 敵デバフスキル | 敵AIのパラメータを弱める効果 | 探索ノード数削減・手のランク制限などが該当 |
| 戦形完成アイテム／スキル | 対局開始時にプレイヤー側が指定の囲いの完成形からスタートできる効果 | `items.json` の `type: "formation_start"` に対応 |
| 夜古火姫（やこびひめ） | 行列式から生まれた行列式の神 | ヤコブ・ヤコビと古事記の八上比売にちなむ本編の中心人物 |

## 表記ゆれ防止のための統一ルール

- コードとマスタ上の分類名は **「戦形（formation）」** に統一する。ただし表示上の「囲い」は
  玉を守る配置だけを指し、攻め方を含む「戦形」と同義にはしない
- 「待った」の英語コード上の識別子は `undo` に統一する
- 「復活の呪文」の英語コード上の識別子は `save_code` に統一する
