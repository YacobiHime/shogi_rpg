# 用語集 (GLOSSARY)

将棋用語とゲーム内用語の対応・簡単な解説をまとめます。将棋に詳しくない協力者や、
実装時に用語のブレを防ぐための参照資料です。

## 将棋の基本用語

| 用語 | 説明 |
|---|---|
| 囲い（かこい） | 玉を守るための駒の配置形。美濃囲い・矢倉・穴熊など |
| 戦形（せんけい） | 対局全体の駒組みの方針（囲い＋戦法の組み合わせ）。本プロジェクトでは「囲い」とほぼ同義で扱う |
| 定跡（じょうせき） | 序盤の指し方として研究・蓄積されてきた手順 |
| 駒落ち（こまおち） | 実力差を埋めるため、上手（弱い側）に対して下手（強い側）が特定の駒を外して対局する形式。飛車落ち・角落ち等 |
| 持ち駒（もちごま） | 相手から取った、盤上に再度打てる駒 |
| 成り（なり） | 駒が敵陣に入った際、裏返って能力が変化すること |
| 詰み（つみ） | 玉がどのように応じても捕獲を避けられない状態。対局の終了条件 |
| 持将棋（じしょうぎ） | 両者の玉が敵陣に入る等して勝敗がつかないまま引き分けとなるルール |
| SFEN | 将棋の局面を表すテキスト形式。チェスのFENに相当 |
| USI | Universal Shogi Interface。将棋エンジンとGUI間の通信プロトコル |
| NNUE | 将棋・チェス等のAIで使われる軽量なニューラルネット型評価関数の方式 |
| MultiPV | エンジンが最善手だけでなく、上位N個の候補手を同時に出力する機能 |
| 次善手（じぜんしゅ） | 最善手の次に良いとされる手 |

## ゲーム内用語（本プロジェクト固有）

| 用語 | 対応する仕組み | 備考 |
|---|---|---|
| 復活の呪文 | セーブデータのエンコード文字列 | ドラクエ由来の呼称。進行状況を英数字文字列化したもの |
| 棋神の巻物（仮） | 最善手ヒントアイテム | `items.json` の `type: "hint"` に対応 |
| 待った | 直前の指し手取り消し機能 | `items.json` の `type: "undo"` に対応。将棋用語としても「待った」で通じる |
| 拠点 | RPG進行上の準備・買い物・装備変更を行う場所 | ティラノスクリプト側のシーンとして実装 |
| 章（チャプター） | RPGのストーリー進行単位 | セーブデータの `chapter` に対応 |
| 敵デバフスキル | 敵AIのパラメータを弱める効果 | 推論時間短縮・手のランク制限などが該当 |
| 戦形完成アイテム／スキル | 対局開始時にプレイヤー側が指定の囲いの完成形からスタートできる効果 | `items.json` の `type: "formation_start"` に対応 |

## 表記ゆれ防止のための統一ルール

- 「戦形」と「囲い」は本プロジェクトでは同義として扱い、ドキュメント・コード上は原則 **「戦形（formation）」** に統一する
- 「待った」の英語コード上の識別子は `undo` に統一する
- 「復活の呪文」の英語コード上の識別子は `resurrection_spell` または `save_code` のいずれかに統一する（実装開始時に確定し、本ファイルに追記すること）
