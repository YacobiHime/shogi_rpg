# 仕様書

## 1. ゲーム概要

将棋の対局をコアバトルとするブラウザRPG。プレイヤーはRPGの物語を進めながら、
使用できる戦形・アイテム・スキルを増やし、段階的に強くなる敵将棋AIに挑む。

## 2. 対応環境

| 項目 | 内容 |
|---|---|
| 必須動作環境 | PC、Google Chrome |
| 努力目標 | スマートフォン（Chrome等） |
| 配信方法 | 無料の静的ホスティング（サーバーサイド処理なし） |

## 3. 難易度仕様

対局UIを通常起動した場合は準備画面を表示し、対戦相手・戦形・難易度・スキルを選んでから
対局を開始する。URLクエリで対局条件が渡された場合は、外部シーンとの連携や
ブックマークからの直接起動のため、準備画面を省略する。

### 3.1 種類

- やさしい
- ふつう
- むずかしい

### 3.2 仕様

- ゲーム開始時に選択
- **ゲーム中いつでも変更可能**（メニューから随時変更できる）
- 難易度は敵AIのパラメータ（探索ノード数・手のランク制限等）への係数として作用する
- 「ふつう」では敵が指すたびに探索ノード数を基準100%・標準偏差45%の正規分布を基に補正し、
  強すぎる着手を抑えるため上振れ幅だけを半分に圧縮して、10〜145%の範囲に制限する
- 進行状況（章・レベル・解禁要素）には影響しない。純粋に対局の難しさのみを変更する

## 4. 勝敗・リトライ仕様

### 4.1 敗北時のフロー

1. 対局に敗北する
2. 以下のいずれかを選択する
   - **その場でリトライ**: 同一条件で即座に再挑戦
   - **拠点に戻って準備**: アイテム収集・スキル装備・戦形変更・難易度変更を行ってから再挑戦
3. 何度でも再挑戦可能（ゲームオーバーによる進行不能は発生しない）

### 4.2 勝利時のフロー

1. 対局に勝利する
2. 経験値・アイテム等の報酬を獲得する
3. 章が進行し、物語が展開する
4. 条件を満たすとレベルアップし、新たな戦形・アイテム・スキルが解禁される

## 5. アイテム・スキル仕様

| 種別 | 名称 | 効果 |
|---|---|---|
| 開始局面変更 | 駒落ちアイテム | 敵の特定の駒を対局開始前に取り除いた状態で開始する |
| 開始局面変更 | 戦形完成アイテム／スキル | プレイヤー側の駒組みが指定の戦形（囲い）の完成形になった状態で対局開始 |
| 開始局面変更 | 持ち駒増加 | プレイヤーの持ち駒を対局開始時から増やした状態にする |
| 対局中補助 | 最善手ヒント | 将棋AIが計算した最善手を一定回数まで表示できる |
| 対局中補助 | 待った | 直前の指し手を一定回数まで取り消せる |
| 敵デバフ | 探索量削減 | 敵AIが探索できるノード数を減らす |
| 敵デバフ | 次善手／N番手デバフ | 敵AIが最善手ではなく、次善手やN番目に有力な手までしか選択しないようにする |

探索量削減スキルは対局準備画面で1つ装備できる。最初の恒久スキル「読み筋封じ」は、
難易度補正後を含む敵の実効探索ノード数を50%にする。未装備も選択できる。

ヒントはプレイヤーの手番でのみ使用でき、現局面を将棋AIに探索させて最善手を表示する。
待ったは敵の応手後に使用でき、直前のプレイヤーの指し手と敵の応手を取り消して
プレイヤーが指し直せる局面へ戻す。スタンドアロン対局の初期上限はヒント2回、待った3回とする。

### 5.1 解禁条件

- 各アイテム・スキル・戦形は、レベルアップまたは章の進行に応じて解禁される
- ヒント回数・待った回数の上限は、レベルアップに応じて増加する
- 対局準備画面には現在のプレイヤーレベルで解禁済みの戦形・スキルだけを表示する
- 未解禁の戦形・スキルをURLから直接指定しても対局を開始できない
- プレイヤーレベルはlocalStorageのセーブデータを正とする。保存データがない初回だけ、旧URLとの
  互換用`level`クエリを初期値として取り込む（省略時はレベル1）

## 6. 戦形（囲い）仕様

### 6.1 基本仕様

- プレイヤーは対局開始前に、解禁済みの戦形の中から1つを選択する
- 選択した戦形の完成局面から対局が開始される（駒組みの過程は省略する）
- 敵側にも戦形の概念があり、章・敵ごとに使用する戦形が設定されている

### 6.2 相性

- 戦形同士には相性（有利・不利）が設定されており、プレイヤーは敵の使用戦形に対して
  有利な戦形を選んで対局に臨むことができる
- 相性表の具体的な設計（三すくみ的な単純化か、将棋理論に基づく段階的な有利不利か）は別途検討する

## 7. 対局仕様

### 7.1 基本ルール

- 通常の将棋のルールに準拠する（合法手判定・王手判定・詰み判定・持将棋判定等）
- 開始局面は「初期配置」ではなく、選択した戦形・駒落ち設定を反映した局面から始まる場合がある

#### 入玉宣言

- 対エンジンの一人用対局では、両者の合意を必要とする24点法の持将棋ではなく、
  27点法の入玉宣言を採用する
- 宣言側の手番で、玉が敵陣3段目以内にあり、王手されていないこと
- 宣言側の敵陣3段目以内の駒が、玉を除いて10枚以上あること
- 玉を除く宣言側の持ち駒と敵陣3段目以内の駒を、飛・角5点、その他1点で数え、
  先手は28点以上、後手は27点以上あること
- プレイヤーは条件成立時に「入玉宣言」ボタンで勝利を宣言できる
- 将棋エンジンにも同じ27点法を設定し、エンジンの宣言成立時はプレイヤーの敗北とする
- ルール条件は日本将棋連盟
  [「将棋のルールに関するご質問」](https://www.shogi.or.jp/faq/rules/)の
  入玉将棋の宣言法に準拠する

### 7.2 敵AI仕様

敵AIの強さは以下のパラメータの組み合わせで決定される。

| パラメータ | 内容 |
|---|---|
| 評価関数（NNUEファイル） | 自由配布のもののみ使用 |
| ノード数制限 | 章・敵・難易度・プレイヤーのデバフスキルにより変動する強さの主基準 |
| 最大思考時間 | 低速端末で思考が終わらない事態を防ぐ安全上限 |
| 手のランク制限 | 最善手のみか、次善手／N番手まで許容するか |
| 使用可能戦形 | 敵ごとに設定された戦形リストの中から使用 |
| 駒落ち | 章・イベントにより敵側に駒落ちが設定される場合がある |

### 7.3 UI仕様（最低限）

- 盤面表示、駒移動（クリック／タップ操作）
- 成り／不成りの選択
- 持ち駒表示・打つ操作
- 選択中の盤上駒・持ち駒を色で強調し、合法な移動先を盤上に表示。同じ持ち駒を再度クリック／タップすると選択解除
- 盤と駒は外部画像へ依存しない自作SVGで描画し、筋・段の座標、成駒の朱文字、
  駒種別に集約した持ち駒の枚数を表示する
- PCを基準としつつ、狭い画面では盤面を横幅へ縮小し、操作ボタンを折り返して表示する
- 投了ボタン
- ヒント使用ボタン（残数表示）
- 待ったボタン（残数表示）
- 詰み・持将棋等の自動判定と結果表示

### 7.4 展示運用時の復旧

- 対局中は確認ダイアログ付きの「最初から」で、同じURL条件の対局を再開始できる
- 結果画面から同じ戦形・敵・難易度でもう一度対局できる
- 初期化に失敗した場合は原因を表示し、その画面から再試行できる
- オフライン時は対局を妨げずに案内を表示し、通信回復後の再読み込みを促す
- 対応ブラウザでは対局中の画面スリープを可能な範囲で抑止する

### 7.5 ノベル連携

- ティラノスクリプトのシナリオから、敵・戦形・難易度・装備スキルを指定して対局を開始できる
- 対局UIは全画面iframeで表示し、終局時に勝敗・終了理由・手数をノベルへ返す
- 勝利時だけ撃破済み敵と章分岐フラグを保存し、敗北・引き分けでは進行を進めない
- 対局iframeの再読込と同一条件リトライでは進行を更新せず、最終結果を受け取った時点で一度だけ反映する
- スタンドアロン起動ではノベル向け結果送信と撃破済み敵の更新を行わない

## 8. セーブ仕様（復活の呪文）

### 8.1 保存対象データ

- 章の進行状況
- プレイヤーレベル
- 解禁済み戦形・アイテム・スキル
- 撃破済みボスの状況
- 現在の難易度設定

### 8.2 保存対象外データ

- 対局途中の局面（対局は章の区切りごとに毎回最初から行う想定のため、対局途中のセーブは行わない）

### 8.3 動作仕様

- 通常時は localStorage に自動セーブされる
- 初回起動時にレベル解禁とヒント・待ったの初期所持数を含むセーブを作成する
- 難易度変更とヒント・待ったの使用は直ちに自動保存し、再読み込み後も残数を引き継ぐ
- 旧形式は現行形式へ移行し、欠損は補完する。破損・未知ID・未来バージョンは警告を表示して
  安全な初期状態へ復旧する
- プレイヤーは任意のタイミングでメニューから「復活の呪文」を文字列として発行できる
- タイトル画面等から呪文を入力することで、対応する進行状況から再開できる
- 入力ミスや文字列の破損はチェックサムにより検出し、エラーメッセージを表示する
- 呪文の文字セットは、判別しにくい文字（`0`と`O`、`1`と`I`等）を除いた英数字とする
- 呪文は16文字（画面表示は4文字区切り）とし、1文字の入力ミスをCRC-8で検出する

## 9. 素材仕様

| 種別 | 方針 |
|---|---|
| 駒・盤画像 | フリー素材（CC BY-SA等）を優先使用。クレジット表記が必要な場合は明記する |
| キャラクター・背景等 | フリー素材を優先探索し、不足分は自作する |
| 評価関数ファイル | 自由配布が明言されているもの（リゼロ評価関数等）のみ使用 |
