# 仕様書

## 1. ゲーム概要

将棋の対局をコアバトルとするブラウザRPG。プレイヤーはRPGの物語を進めながら、
使用できる戦形・アイテム・スキルを増やし、段階的に強くなる敵将棋AIに挑む。

製品版では、夜古火姫（やこびひめ）という神様に導かれ、将棋の基本、棒銀、四間飛車、
右四間飛車、居飛車、穴熊を順に学び、最後に名人へ挑戦する。物語・学習内容・素材差し替えの
詳細な目標は [本編ゲーム設計（叩き台）](./GAME_DESIGN.md) に定める。

「敵を倒す」は友好的な将棋勝負に勝つことを意味し、人物を傷つける表現にはしない。
主人公の名前はゲーム開始時に入力し、ブラウザ内のセーブデータへ保存する。
主人公は立ち絵と発話台詞を持たず、選択肢と1〜2文の簡潔な内心で表現する。

## 2. 対応環境

| 項目 | 内容 |
|---|---|
| 必須動作環境 | PC、Google Chrome |
| 努力目標 | スマートフォン（Chrome等） |
| 配信方法 | 無料の静的ホスティング（サーバーサイド処理なし） |

## 3. 難易度仕様

対局UIを通常起動した場合は準備画面を表示し、対戦相手・戦形・難易度・スキルを選んでから
対局を開始する。URLクエリで対局条件が渡された場合は、外部シーンとの連携や
ブックマークからの直接起動のため、準備画面を省略する。

### 3.1 種類

- やさしい
- ふつう
- むずかしい

### 3.2 仕様

- ゲーム開始時に選択
- **ゲーム中いつでも変更可能**（メニューから随時変更できる）
- 難易度は敵AIのパラメータ（探索ノード数・手のランク制限等）への係数として作用する
- 「ふつう」では敵が指すたびに探索ノード数を基準100%・標準偏差45%の正規分布を基に補正し、
  強すぎる着手を抑えるため上振れ幅だけを半分に圧縮して、10〜145%の範囲に制限する
- 進行状況（章・レベル・解禁要素）には影響しない。純粋に対局の難しさのみを変更する
- 難易度によって経験値、通貨、アイテム、物語進行へ差を付けない

## 4. 勝敗・リトライ仕様

### 4.1 敗北時のフロー

1. 対局に敗北する
2. 以下のいずれかを選択する
   - **その場でリトライ**: 同一条件で即座に再挑戦
   - **拠点に戻って準備**: アイテム収集・スキル装備・戦形変更・難易度変更を行ってから再挑戦
3. 何度でも再挑戦可能（ゲームオーバーによる進行不能は発生しない）

### 4.2 勝利時のフロー

1. 対局に勝利する
2. 経験値・アイテム等の報酬を獲得する
3. 通常敵なら本道の次地点、ボスなら次の章が解禁され、物語が展開する
4. 条件を満たすとレベルアップし、新たな戦形・アイテム・スキルが解禁される

### 4.3 村内コースの進行

- 全7章とし、1つの村を1章として扱う
- 各村は、入口から通常敵を順番に倒し、最後にボスへ挑む1本の本道を持つ
- 本道の次地点は、手前の通常敵へ勝利した時点で解禁する。ボスはその村の通常敵を
  すべて倒した後に解禁し、ボス撃破で次章を解禁する
- 宝箱は本道から分かれる任意の寄り道とし、未開封でも本道とボス戦を進められる
- 現在まで連続して踏破した本道の地点に、主人公と夜古火姫の進行マーカーを表示する
- 頼まれごとと棋具店はコースの下部へまとめる。棋具店はコース進行と独立して利用でき、
  頼まれごとの報告可否は従来どおり各クエストの対局条件から判定する
- 通常対局とボス対局は、それぞれ初回勝利時だけ経験値を与える
- 初回勝利の判定には対局開始前の進行状態を使い、埋め込み対局が終局時に撃破済みIDを
  先行保存しても、初回報酬とボス撃破による次章解禁を一度だけ適用する
- 起動時は撃破済みボスから解禁済みの最新章を復元し、章番号だけが古いセーブを補正する
- コース導入前のセーブに順序を飛ばした通常敵の撃破実績がある場合、その実績と報酬は維持するが、
  進行マーカーと次の目的地は最初の未勝利地点へ戻し、未クリアの本道を飛ばせないようにする
- コース導入前に既に倒したボスは救済対象とし、章クリア、次章解禁、旧村での踏破表示を維持する
- 再戦では経験値を与えず、同じ相手だけを繰り返すレベル上げを必須にしない
- ボス撃破後も以前の村へ戻り、未完了の対局、宝箱、定跡書、サブクエストを回収できる
- 各村には地点選択式の探索、店、ゲーム内通貨「棋貨」、宝箱、任意サブクエストを用意する
- 新規開始からエンディングまでの本道は、7村の全25対局を順に攻略する構成とする

## 5. アイテム・スキル仕様

| 種別 | 名称 | 効果 |
|---|---|---|
| 開始局面変更 | 駒落ちアイテム | 敵の特定の駒を対局開始前に取り除いた状態で開始する |
| 開始局面変更 | 戦形完成アイテム／スキル | プレイヤー側の駒組みが指定の戦形（囲い）の完成形になった状態で対局開始 |
| 開始局面変更 | 持ち駒増加 | プレイヤーの持ち駒を対局開始時から増やした状態にする |
| 対局中補助 | 上位3手ヒント | 将棋AIが計算した最善手・次善手・3番手を一定回数まで表示できる |
| 対局中補助 | 待った | 直前の指し手を一定回数まで取り消せる |
| 敵デバフ | 探索量削減 | 敵AIが探索できるノード数を減らす |
| 敵デバフ | 次善手／N番手デバフ | 敵AIが最善手ではなく、次善手やN番目に有力な手までしか選択しないようにする |

探索量削減スキルは対局準備画面で1つ装備できる。最初の恒久スキル「読み筋封じ」は、
難易度補正後を含む敵の実効探索ノード数を50%にする。未装備も選択できる。

ヒントはプレイヤーの手番でのみ使用でき、敵の強さや難易度とは独立して現局面を探索する。
プレイヤーが停止するまで読み続け、最善手・次善手・3番手と各評価値、探索深さ、探索局面数を
リアルタイムで更新する。各候補には自分と相手の応手を含む6手先までの読み筋も表示する。
指し手は「5二銀」「5五歩打」のように、筋を数字、段を漢数字、駒名を日本語で表す。解析中に
ヒントボタンを再度押すと停止し、タブが非表示になった場合もCPU負荷を残さないため自動停止する。
待ったは敵の応手後に使用でき、直前のプレイヤーの指し手と敵の応手を取り消して
プレイヤーが指し直せる局面へ戻す。スタンドアロン対局の初期上限はヒント2回、待った3回とする。

### 5.1 解禁条件

- 各アイテム・スキル・戦形は、レベルアップまたは章の進行に応じて解禁される
- 次章の攻略に必要な戦形と基本ガイドは章解禁時に付与し、本道の進行をレベル不足で止めない
- ヒント回数・待った回数の上限は、レベルアップに応じて増加する
- 対局準備画面には現在のプレイヤーレベルで解禁済みの戦形・スキルだけを表示する
- 未解禁の戦形・スキルをURLから直接指定しても対局を開始できない
- プレイヤーレベルはlocalStorageのセーブデータを正とする。保存データがない初回だけ、旧URLとの
  互換用`level`クエリを初期値として取り込む（省略時はレベル1）

## 6. 戦形（囲い）仕様

### 6.1 基本仕様

- プレイヤーは対局開始前またはシナリオ上の指定により、解禁済みの戦形から使用方針を選択する
- 本編の学習対局は原則として平手初期局面から開始し、囲いと攻めの駒組みを矢印ガイドで学ぶ
- 戦形完成局面から開始する機能は、練習局面、救済用アイテム、特別対局に限って使用する
- 敵側にはプレイヤー用の開始戦形とは独立した「敵専用戦法定跡」を設定し、章・敵ごとの
  攻め方と囲いを平手初期局面から優先して組ませる

### 6.2 相性

- 戦形同士には相性（有利・不利）が設定されており、プレイヤーは敵の使用戦形に対して
  有利な戦形を選んで対局に臨むことができる
- 相性表の具体的な設計（三すくみ的な単純化か、将棋理論に基づく段階的な有利不利か）は別途検討する

## 7. 対局仕様

### 7.1 基本ルール

- 通常の将棋のルールに準拠する（合法手判定・王手判定・詰み判定・持将棋判定等）
- 開始局面は「初期配置」ではなく、選択した戦形・駒落ち設定を反映した局面から始まる場合がある

#### 入玉宣言

- 対エンジンの一人用対局では、両者の合意を必要とする24点法の持将棋ではなく、
  27点法の入玉宣言を採用する
- 宣言側の手番で、玉が敵陣3段目以内にあり、王手されていないこと
- 宣言側の敵陣3段目以内の駒が、玉を除いて10枚以上あること
- 玉を除く宣言側の持ち駒と敵陣3段目以内の駒を、飛・角5点、その他1点で数え、
  先手は28点以上、後手は27点以上あること
- プレイヤーは条件成立時に「入玉宣言」ボタンで勝利を宣言できる
- 将棋エンジンにも同じ27点法を設定し、エンジンの宣言成立時はプレイヤーの敗北とする
- ルール条件は日本将棋連盟
  [「将棋のルールに関するご質問」](https://www.shogi.or.jp/faq/rules/)の
  入玉将棋の宣言法に準拠する

### 7.2 敵AI仕様

敵AIの強さは以下のパラメータの組み合わせで決定される。

| パラメータ | 内容 |
|---|---|
| 評価関数（NNUEファイル） | 自由配布のもののみ使用 |
| ノード数制限 | 章・敵・難易度・プレイヤーのデバフスキルにより変動する強さの主基準 |
| 最大思考時間 | 低速端末で思考が終わらない事態を防ぐ安全上限 |
| 手のランク制限 | 最善手のみか、次善手／N番手まで許容するか |
| 許可開始戦形 | プレイヤーがその敵との対局へ持ち込める開始戦形のリスト。敵自身の戦法指定とは別 |
| 敵専用戦法定跡 | `enemy_openings.json`の手順と完成条件を敵ごとに指定し、通常の自己定跡より優先 |
| 駒落ち | 章・イベントにより敵側に駒落ちが設定される場合がある |

敵専用戦法定跡の割り当ては次のとおりとする。

| 章 | 敵の戦法 |
|---:|---|
| 1 はじまりの町 | 専用指定なし。通常のAI方針を使用 |
| 2 棒銀村 | 全敵が後手棒銀 |
| 3 四間飛車村 | 全敵が後手四間飛車＋美濃囲い |
| 4 右四間＋エルモ囲い村 | 全敵が角道を止める後手四間飛車 |
| 5 居飛車村 | カクは角換わり、ヨコは横歩取り、アイは相掛かり、ボスは居飛車 |
| 6 穴熊村 | 全敵が後手四間飛車穴熊 |
| 7 名人村 | 専用指定なし。通常のAI方針を使用 |

- 敵専用戦法定跡がある対局ではエンジンの標準自己定跡を無効にし、現局面で合法な
  定跡候補だけをUSIの`searchmoves`へ渡して、探索強度を保ったまま戦法を優先する
- 王手への応手、駒の捕獲、進路妨害等により次の定跡手が違法になった場合は、候補を強制せず
  通常の合法手探索へフォールバックし、対局を停止させない
- 四間飛車系の定跡では飛車を振った後、合法手候補を1〜4筋に飛車を維持する手へ制約する。
  制約下に合法手がない場合も通常探索へフォールバックする
- ローカル検証用の振電3 `nn.bin`はHalfKP512で、現行Web版WASMが読むHalfKP256とは互換性がない。
  さらに第三者再配布許諾も未確認のため、マスタ、Hostingビルド、配布物へ含めない。振り飛車の
  暫定的な維持は、内蔵評価と敵専用戦法定跡、`searchmoves`制約で実現する

### 7.3 UI仕様（最低限）

- 盤面表示、駒移動（クリック／タップ操作）
- 成り／不成りの選択
- 持ち駒表示・打つ操作
- 選択中の盤上駒・持ち駒を色で強調し、合法な移動先を盤上に表示。同じ持ち駒を再度クリック／タップすると選択解除
- 敵の直前手は移動元を黄色、移動先を橙色で盤上に表示する。駒打ちは移動先だけを表示し、
  待ったで局面を戻した場合は直前手の表示を消す
- 盤と駒は外部画像へ依存しない自作SVGで描画し、筋を数字、段を漢数字とする座標、成駒の朱文字、
  駒種別に集約した持ち駒の枚数を表示する
- PCを基準としつつ、狭い画面では盤面を横幅へ縮小し、操作ボタンを折り返して表示する
- 投了ボタン
- ヒント使用ボタン（残数表示）
- 待ったボタン（残数表示）
- 詰み・持将棋等の自動判定と結果表示

### 7.4 展示運用時の復旧

- 対局中は確認ダイアログ付きの「最初から」で、同じURL条件の対局を再開始できる
- 結果画面から同じ戦形・敵・難易度でもう一度対局できる
- 初期化に失敗した場合は原因を表示し、その画面から再試行できる
- オフライン時は対局を妨げずに案内を表示し、通信回復後の再読み込みを促す
- 対応ブラウザでは対局中の画面スリープを可能な範囲で抑止する

### 7.5 ノベル連携

- ティラノスクリプトのシナリオから、敵・戦形・難易度・装備スキルを指定して対局を開始できる
- 対局UIは全画面iframeで表示し、終局時に勝敗・終了理由・手数をノベルへ返す
- 勝利時だけ撃破済み敵と章分岐フラグを保存し、敗北・引き分けでは進行を進めない
- 対局iframeの再読込と同一条件リトライでは進行を更新せず、最終結果を受け取った時点で一度だけ反映する
- スタンドアロン起動ではノベル向け結果送信と撃破済み敵の更新を行わない

### 7.6 学習ガイドと定跡書

- 夜古火姫が、移動元から移動先への矢印、注目駒とマスの強調、短い解説で学習を案内する
- 対局盤の横（狭い画面では盤の下）にやこび姫のミニキャラと吹き出しを表示する
- 盤面のSFENから先手・後手の駒配置を確認し、棒銀、金矢倉囲い、右四間飛車を認識した時点で
  `formation_callouts.json`の文言を1局につき各1回だけ発話する
- 学習ガイドは、決められた局面を学ぶ講座、定跡書に基づく序盤方針、AIによる最善手ヒントを区別する
- 定跡書は消耗せず、一度入手すると定跡集から何度でも盤面付きで見直せる
- 所持する定跡書は本編対局でも選択でき、対応する序盤ガイドを表示できる
- 定跡ガイドの閲覧では最善手ヒントの残回数を消費しない
- 矢印と強調は色だけに依存せず、線種、記号、ラベルを併用する
- 講座で指定外の手を指した場合は、夜古火姫が理由を説明して自動で局面を戻す。この処理では待ったを消費しない
- 定跡書は代表手順8〜16手程度、主要な応手2〜3分岐を基本とする

## 8. セーブ仕様（復活の呪文）

### 8.1 保存対象データ

- プレイヤー名と呼ばれ方
- 章の進行状況
- プレイヤーレベル
- 経験値とゲーム内通貨
- 解禁済み戦形・アイテム・スキル
- 各対戦相手の初回勝利、撃破済みボスの状況
- 入手済み定跡書、開封済み宝箱、サブクエスト進行、アイテム所持数
- 現在の難易度設定

### 8.2 保存対象外データ

- 対局途中の局面（対局は章の区切りごとに毎回最初から行う想定のため、対局途中のセーブは行わない）

### 8.3 動作仕様

- 通常時は localStorage に自動セーブされる
- セーブ、経験値計算、復活の呪文の生成・復元にサーバー通信を使わない
- 初回起動時にレベル解禁とヒント・待ったの初期所持数を含むセーブを作成する
- 難易度変更とヒント・待ったの使用は直ちに自動保存し、再読み込み後も残数を引き継ぐ
- 旧形式は現行形式へ移行し、欠損は補完する。破損・未知ID・未来バージョンは警告を表示して
  安全な初期状態へ復旧する
- 旧セーブの撃破済み敵は削除しない。通常敵の撃破順に空きがある場合は、最初の未勝利地点を
  次の目的地として表示し、既に撃破済みのボスによる章クリアだけは維持する
- プレイヤーは任意のタイミングでメニューから「復活の呪文」を文字列として発行できる
- タイトル画面等から呪文を入力することで、対応する進行状況から再開できる
- 入力ミスや文字列の破損はチェックサムにより検出し、エラーメッセージを表示する
- 呪文の文字セットは、判別しにくい文字（`0`と`O`、`1`と`I`等）を除いた英数字とする
- 現行の16文字の呪文は旧形式として引き続き読み込めるようにする
- 本編用の新形式は、追加された進行情報とプレイヤー名を含められる版番号・チェックサム付きの
  可変長文字列とし、画面からコピーしやすい区切りとコピー操作を用意する

## 9. 素材仕様

| 種別 | 方針 |
|---|---|
| 駒・盤画像 | フリー素材（CC BY-SA等）を優先使用。クレジット表記が必要な場合は明記する |
| キャラクター・背景等 | フリー素材を優先探索し、不足分は自作する |

- キャラクター、背景、UI、BGM、効果音、任意ボイスは素材マスタを介して参照し、

## 12. 七つの将棋村 本編仕様

- 本編入口は`src/rpg/`のデータ駆動SPAとし、通常の将棋対局UIを同一オリジンiframeで開く
- `data/world.json`の7章・25対局を巡り、各村のボス撃破で次章を解禁する
- 各村の通常敵を本道の順番で全員倒すとボス地点が解禁され、ボス撃破で次章へ進む
- 宝箱は本道から分岐する任意地点とし、頼まれごとと棋具店はコース下部の施設として表示する
- 主人公と夜古火姫の進行マーカー、次の目的地、解禁済み・勝利済み・開封済み状態をコース上に表示する
- 初回勝利だけ経験値・棋貨・定跡書・アイテム報酬を得て、再戦では少額の棋貨だけを得る
- 店、宝箱、サブクエスト、定跡集、難易度・装備設定、復活の呪文を本編画面から利用できる
- 消費型の敵弱体化棋具は装備後の対局開始時に1個消費し、探索量または候補手ランクへ反映する
- 解禁済みの旧村へいつでも戻れ、第7章の伝説の名人に勝利した後も再戦と任意要素の回収ができる
- 夜古火姫は旅と将棋学習の案内役とし、駒の人格や対局をまたぐ軍勢・損耗は本編仕様に含めない
